# Lean Canvas — Bookshelve

> Produced by: `facilitating-inception` Phase 1 (po-agent)

## Problem
1. Readers lose track of books they want to read — recommendations get forgotten
2. No lightweight way to mark progress without a full Goodreads account
3. Reading lists scattered across notes apps, browser bookmarks, and memory

## Customer Segments
- **Primary:** Avid readers aged 25–45 who get book recommendations from friends and podcasts
- **Secondary:** Casual readers who finish 4–10 books a year and want a simple tracker

## Unique Value Proposition
The fastest way to save a book recommendation and know what to read next.

## Solution
1. One-tap save from a book title search
2. Reading list with a single "Mark as read" action
3. No account required to start — local first, sync later

## Channels
- Word of mouth via sharing reading lists
- SEO on book title pages

## Revenue Streams
- Free tier: unlimited list, no sync
- Pro tier ($3/month): sync across devices, reading stats

## Cost Structure
- Hosting (minimal — mostly static)
- Search API calls (ISBNdb or Open Library)

## Key Metrics
- Books saved per user per week
- % users who return within 7 days
- Free-to-pro conversion rate

## Unfair Advantage
Radically simple — no social features, no ratings, no reviews. Just save and read.
